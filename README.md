# portal-app-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常作为门户应用的辅助界面、说明文档或轻量工具使用，不针对任何特定域名或具体网站。

## 项目简介

portal-app-guide 是一个静态页面集合，旨在提供通用、可复用的 HTML 资源。每个页面独立自包含，不依赖外部服务或数据库，可直接在浏览器中打开或通过静态托管部署使用。

## 目录说明

```
.
├── pages/           # 存放所有独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/          # 页面所需的静态资源（如 CSS、JS、图片等）
│   ├── styles/
│   └── scripts/
├── README.md        # 本文件
└── LICENSE          # 许可证文件（如有）
```

- **pages/**：核心目录，每个 HTML 文件为一个独立页面。文件名应反映页面用途，便于识别。
- **assets/**：存放页面引用的共用资源，按类型分类组织。

## 页面归档说明

本仓库中的 HTML 页面遵循以下规范：

- 每个页面均为独立、自包含的文件，不依赖外部 API 或特定后端。
- 页面内容与样式尽量简洁，避免冗余依赖。
- 页面标题、描述及元数据应清晰标注其用途。
- 若页面包含交互功能，建议使用纯前端实现（如 JavaScript 本地处理）。

## 维护说明

- 新增页面请放入 `pages/` 目录，并确保文件名不重复。
- 更新页面时，请保持向后兼容或明确标注版本变更。
- 共用资源（CSS/JS 等）请统一放置在 `assets/` 目录下，避免在页面内直接嵌入大量样式或脚本。
- 提交前建议检查页面在主流浏览器中的基本显示效果。
- 欢迎提交 Pull Request 添加新页面或改进现有内容。
